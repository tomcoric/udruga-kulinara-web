Kako implementirati stranicu?

Isti dizajn, različita tehnička rješenja — odaberi što ti odgovara

A

Jedna HTML datoteka (sve u jednom)

HTML + CSS + JS sve u jednoj datoteci index.html. Otvoriš u browseru — odmah radi. Najlakše za pregledavati i dijeliti. Idealno ako je ovo prototip ili prezentacija klijentu.

Za

  • Jedan klik za otvaranje
  • Nema dependency-ja
  • Lako poslati mailom/ZIP-om

Protiv

  • Teže održavati ako postane veliko
B

HTML + CSS + JS (odvojene datoteke)

Standardna web struktura: index.html, style.css, main.js. Čistije za razvoj, lakše za kasniju integraciju u WordPress kao child theme.

Za

  • Čišći kod, lakše uređivanje
  • Spreman za WordPress integraciju
  • Standardna praksa

Protiv

  • Potreban lokalni server za pregled (Live Server u VSCode)
C

WordPress page template (PHP)

Direktno kao WordPress page-template.php s inline CSS-om. Možeš uploadati na server i zamijeniti trenutnu početnu stranicu bez rebuilda cijele teme.

Za

  • Odmah ide na WordPress
  • Nema potrebe za child theme

Protiv

  • Potreban FTP/hosting pristup za pregled
  • PHP sintaksa

💡 Preporuka: B ako planiraš integrirati u WordPress, A ako želiš brzo vidjeti rezultat odmah.
